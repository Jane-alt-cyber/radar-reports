test-time compute scaling 测试时计算扩展
narrativenew🔁 连续 7 天
让模型在回答时使用更多计算资源来提升答案质量的技术路径。
是什么

Test-time compute scaling 是一种新的 AI 能力提升范式。传统的做法是在训练时投入更多计算资源来提升模型能力，而这种方法是在模型已经训练完成后，在实际使用（推理）时让模型"思考更久"来得到更好的答案。

具体来说，当用户提出一个复杂问题时，模型会分配更多计算步骤来分析问题、探索不同的解题路径、验证答案的正确性。这类似于人类在面对难题时会花更多时间思考，而不是立即给出答案。这种方法特别适用于需要复杂推理的任务，如数学证明、代码调试、科学问题分析等。

这个概念的重要性在于它开辟了一条与传统"训练更大模型"不同的技术路线，为 AI 能力提升提供了新的可能性。

为什么现在火

这个概念在近期受到关注主要有几个原因：首先，传统的训练时扩展（training-time scaling）面临边际效应递减和成本急剧上升的挑战，业界需要寻找新的能力提升路径。其次，推理时计算的技术基础已经成熟，包括更高效的推理框架和硬件支持。

此外，一些实验结果显示测试时计算扩展在特定任务上能带来显著的性能提升，这为这一技术路线提供了实证支持。随着 AI 应用场景越来越复杂，用户对准确性的要求也在提高，愿意为更好的答案等待更长时间。

谁在推动

目前推动这一概念的主要是学术研究机构和一些技术驱动的 AI 公司。具体的关键玩家信息有限，但可以观察到这一概念在 AI 研究社区中获得了越来越多的讨论。

一些专注于推理优化的技术公司和研究实验室正在探索相关技术，包括如何在保持推理效率的同时实现计算扩展，以及如何设计更好的推理时优化策略。

Physical AI 物理AI
narrativenew🔁 连续 7 天
让AI模型控制和操作物理世界中的机器人、车辆等实体设备的技术路径。
是什么

Physical AI（物理AI）是指将AI模型的推理能力直接嵌入到物理世界的机器和设备中，让这些设备能够自主感知环境、做出决策并执行物理动作。与传统的云端AI不同，Physical AI强调AI推理在边缘设备上的实时运行，包括机器人、自动驾驶汽车、工业设备等。

这个概念的核心在于将AI从纯数字空间扩展到物理空间，让AI不仅能处理文本、图像等数字信息，还能通过传感器感知物理世界，通过执行器影响物理世界。每个智能设备都将成为一个独立的AI推理节点，形成分布式的智能网络。

从技术架构看，Physical AI需要解决延迟敏感的实时推理、多模态感知融合、安全可靠的决策执行等核心问题，这对AI芯片的算力密度、功耗控制和专用架构提出了新要求。

为什么现在火

多个技术和商业因素在2024-2025年集中爆发。首先，大语言模型的多模态能力突破让AI具备了理解复杂物理场景的基础，而机器人领域的具身智能（Embodied AI）研究也取得关键进展。

其次，AI芯片的边缘推理能力快速提升，功耗和成本持续下降，让复杂AI模型能够在移动设备和嵌入式系统中实时运行。特别是专用AI芯片的成熟，为Physical AI的大规模部署创造了硬件条件。

商业层面，自动驾驶、工业自动化、服务机器人等领域的市场需求日益迫切，而传统的基于规则或简单神经网络的控制方式已经接近天花板，迫切需要更智能的解决方案。

谁在推动

英伟达是这个概念的主要推动者，将其作为下一代GPU市场的核心战略。英伟达CEO黄仁勋多次强调"Physical AI是下一波浪潮"，并推出了专门的Jetson平台和机器人仿真平台Omniverse。

机器人公司如Boston Dynamics、Tesla（机器人部门）、Figure AI等在具身智能方面持续投入。汽车行业的特斯拉、Waymo等自动驾驶公司也在推动相关技术发展。

芯片公司方面，高通、英特尔、AMD等都在开发专门的边缘AI芯片。中国的地平线、黑芝麻等公司也在汽车AI芯片领域积极布局。

竞对动态

OpenAI：暂无对Physical AI概念的直接公开表态，但其投资的机器人公司1X和Figure的合作显示其对具身智能的关注。

Google：通过DeepMind在机器人学习方面持续投入，RT-2等项目体现了其将大模型应用于机器人控制的技术路线，但未明确使用Physical AI这一术语。

Anthropic：专注于AI安全研究，暂无Physical AI相关的公开表态。

Meta：暂无对Physical AI概念的公开表态，其AI研究更多集中在多模态和社交应用场景。

Project Glasswing（玻璃翼项目）是由 Amazon Web Services 和 Anthropic 共同发起的新倡议，旨在为 AI 时代的关键软件提供安全保障。从项目名称和参与方来看，这个项目聚焦于 AI 系统的软件安全问题，特别是那些对 AI 基础设施至关重要的软件组件。

TeraFab 太拉晶圆厂
infranew
美国政府主导的万亿级超大规模半导体制造设施计划
是什么

TeraFab（Tera-scale Fabrication facility）是美国政府提出的新一代超大规模半导体制造设施概念。"Tera"指代万亿级规模，既指投资规模（万亿美元级），也指产能规模（年产万亿颗芯片）。该概念的核心是将传统的单体晶圆厂升级为集群式、模块化的超大型制造综合体。

根据目前公开信息，TeraFab 计划整合多家头部企业的技术和资源，形成从芯片设计、制造到封测的完整生态链。Intel 提供制造工艺和产线技术，SpaceX 贡献先进材料和精密制造经验，Tesla 则带来自动化生产和能源管理方案。该项目被视为美国重建半导体制造优势、应对地缘政治挑战的关键举措。

为什么现在火

谁在推动

vibecoding 感觉式编程
narrativerising
依靠直觉和感觉而非严格规范进行的编程方式
是什么

Vibecoding 指的是程序员在编写代码时，更多依靠经验直觉、代码"感觉"和上下文理解，而不是严格遵循预定义的编程规范或详细的技术文档。这种编程方式强调开发者对代码流畅性和自然性的感知，类似于"凭感觉"写代码。

在 AI 辅助编程普及的背景下，vibecoding 通常指开发者与 AI 编程助手（如 GitHub Copilot、Cursor 等）协作时采用的一种更加直观、探索性的编程模式。开发者可能会基于对问题的大致理解就开始编码，依靠 AI 的建议和自己的直觉来逐步完善代码，而不是先制定详细的架构设计。

这种方式的支持者认为，现代开发环境和 AI 工具使得"边写边想"成为可能，能够提高开发效率和创造力。批评者则担心这种方式可能导致代码质量下降和技术债务积累。

为什么现在火

AI 编程助手的普及是 vibecoding 概念兴起的主要推动力。随着 GPT-4、Claude 等大模型在代码生成方面的能力提升，程序员发现可以更多地依赖 AI 来填补技术细节，自己专注于高层逻辑和创意实现。

同时，现代软件开发的快速迭代需求也为这种编程方式提供了土壤。在原型开发、概念验证等场景下，vibecoding 被认为能够更快地将想法转化为可运行的代码。

另一个背景是年轻程序员群体的成长，他们从学习阶段就习惯了与 AI 协作编程，对传统的"先设计后编码"模式不那么依赖。

谁在推动

主要推动者是个人开发者和小型技术团队，特别是那些从事快速原型开发、创业项目或个人项目的程序员。一些技术 KOL 和编程教育者也在社交媒体上讨论和推广这个概念。

值得注意的是，vibecoding 更多是一种自下而上的草根现象，还没有被大型科技公司官方认可或推广。一些开源项目维护者和独立开发者是这个概念的主要倡导者。

竞对动态

暂无公开表态。主要科技巨头对 vibecoding 这一具体概念尚未有明确的官方表态，但他们都在积极推进 AI 编程助手产品的发展。

JSIR JavaScript中间表示
toolnew
JavaScript 编程语言的高级中间表示形式，用于保存和优化代码结构信息。
是什么

JSIR（JavaScript Intermediate Representation）是一种介于源代码和机器码之间的代码表示形式。传统的编译器在处理 JavaScript 时往往会丢失抽象语法树（AST）中的部分信息，而 JSIR 通过保留完整的 AST 信息，使得编译器能够进行更精确的高级优化。

这项技术主要解决 JavaScript 执行效率问题。当前 JavaScript 引擎在将源码转换为可执行代码的过程中，由于信息丢失导致优化受限。JSIR 提供了一个中间层，让编译器能够更好地理解代码的语义结构，从而实现更深层次的性能优化。目前这项技术还处于 RFC（Request for Comments）提案阶段，正在征集社区反馈。

为什么现在火

JavaScript 性能优化需求日益迫切。随着 AI 应用前端化趋势明显，浏览器端需要处理越来越复杂的计算任务，包括模型推理、实时音视频处理等。传统 JavaScript 引擎的优化方式已无法满足这些高性能需求。

同时，WebAssembly 的发展也推动了 JavaScript 生态对更好中间表示的需求。开发者希望在不放弃 JavaScript 灵活性的前提下，获得接近原生代码的执行性能。JSIR 概念的提出正好契合了这一技术发展趋势。

谁在推动

目前主要由 JavaScript 引擎开发社区推动，具体的主导机构或公司尚未明确。从 RFC 形式来看，这更像是一个开源社区驱动的技术标准化努力，而非某个特定公司的商业项目。

预计 Mozilla（Firefox）、Google（Chrome）、Microsoft（Edge）等浏览器厂商以及 Node.js 团队会是主要的技术贡献者和标准制定参与方。

Phase-Associative Memory 相位联想记忆
modelnew
使用复数值表示的循环序列模型，通过相位信息进行记忆联想
是什么

Phase-Associative Memory（相位联想记忆，PAM）是一种新型的循环序列模型（recurrent sequence model）架构，其核心特点是所有的数据表示都采用复数值（complex-valued）而非传统的实数值。

在传统的神经网络中，信息通常用实数来表示，而 PAM 引入了复数的概念，利用复数的幅度（magnitude）和相位（phase）两个维度来编码信息。相位信息可以用来表示不同概念之间的关联关系，类似于人脑中记忆的联想机制。这种设计允许模型在处理序列数据时，不仅考虑信息的强度，还能捕捉信息之间的相对关系和依赖模式。

该架构主要应用于需要长期记忆和复杂关联推理的任务，如自然语言处理、时序预测等领域。

为什么现在火

目前关于 PAM 概念火热度的信息有限。从技术发展角度看，这可能与当前 AI 社区对提升模型记忆能力和序列建模效率的需求有关。传统的 Transformer 架构在处理长序列时面临计算复杂度和记忆容量的挑战，而循环结构的复数值模型可能提供了新的解决思路。

Inclusion-of-Thoughts 思维包含法
frameworknew
一种通过净化决策空间来缓解 AI 偏好不稳定性的推理框架。
是什么

Inclusion-of-Thoughts（IoT）是一种新的 AI 推理框架，旨在解决大语言模型在决策过程中的"偏好不稳定"问题。具体来说，当 AI 面对相同或相似问题时，可能会给出前后矛盾的答案或判断，这种不一致性影响了 AI 系统的可靠性。

该框架的核心思路是通过"净化决策空间"（Purifying the Decision Space）来提升推理的稳定性。不同于传统的思维链（Chain-of-Thought）方法，IoT 更强调在推理过程中包含多种可能的思维路径，然后通过特定机制筛选和整合这些路径，最终得出更加稳定和一致的结论。

从技术实现角度看，这是对现有推理方法的改进，试图在保持推理能力的同时提升决策的一致性和可预测性。

为什么现在火

这个概念在 2026年 4月开始受到关注，主要原因是 AI 系统在实际部署中暴露出的偏好不稳定问题日益严重。随着大模型在更多关键决策场景中的应用，用户和开发者都发现了一个普遍问题：同一个 AI 对相似问题的回答可能存在明显矛盾。

此外，监管机构和行业组织对 AI 决策一致性的要求不断提高，特别是在金融、医疗等高风险领域，AI 系统的"情绪化"或"摇摆不定"已经成为阻碍大规模商业化的重要因素。IoT 框架的提出正好回应了这一迫切需求。

谁在推动
